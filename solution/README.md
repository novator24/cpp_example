# Word Frequency Counter

Высокопроизводительная однопоточная утилита на C++ для подсчета частоты слов в текстовом файле, отсортированная по убыванию частоты и затем по алфавиту.

## Особенности реализации
- **Максимальная скорость I/O**: Используется `mmap` для чтения файла целиком в память, что исключает накладные расходы на системные вызовы `read` и копирование буферов.
- **Оптимизированный парсинг**: Предварительно вычисленные lookup-таблицы (массивы) для проверки `is_alpha` и перевода в нижний регистр `to_lower` без ветвлений.
- **Быстрая хэш-таблица**: Кастомная реализация Flat Hash Map с открытой адресацией (linear probing), оптимизированная под конкретную задачу.
- **Arena Allocator**: Все уникальные слова сохраняются в единый непрерывный вектор (`std::vector<char> arena`), что минимизирует количество аллокаций памяти (отсутствуют вызовы `new` или `malloc` на каждое слово).
- **Быстрый хэш**: Используется вариация FNV-1a для быстрого вычисления хэша прямо во время парсинга слова.
- **Оптимизированный вывод**: Используется буферизованный `fwrite` (`fprintf` с `setvbuf` на 64 КБ).

## Требования
- Компилятор с поддержкой C++17 (GCC, Clang)
- CMake 3.10+
- ОС: Linux (протестировано на Debian 11)

## Сборка

```bash
sudo apt update && sudo apt install -y build-essential
g++ --version
mkdir build
cd build
cmake -DCMAKE_CXX_COMPILER=g++ ..
make -j$(nproc)
```

## Использование

```bash
./freq [input_file] [output_file]
```

Пример:
```bash
$ cat in.txt
The time has come, the Walrus said, to talk of many things...
$ ./freq in.txt out.txt
$ cat out.txt
2 the
1 come
1 has
1 many
1 of
1 said
1 talk
1 things
1 time
1 to
1 walrus
```

## Тестирование

Для запуска тестов используйте скрипт на Python:
```bash
cd tests
python3 test_freq.py
```

## Установка через Docker (Для администратора)

Вы можете собрать и запустить утилиту в изолированном контейнере Docker на базе Debian 11.

1. **Сборка образа:**
   ```bash
   docker build -t word-freq .
   ```

2. **Запуск:**
   Для обработки файлов на хост-машине необходимо прокинуть директорию с файлами внутрь контейнера через volume (`-v`).
   ```bash
   # Предположим, in.txt находится в текущей директории
   docker run --rm -v $(pwd):/data word-freq /data/in.txt /data/out.txt
   ```

## CI/CD Процесс

В репозитории настроен процесс непрерывной интеграции (CI) через GitLab CI (`.gitlab-ci.yml`).
Процесс состоит из 3 стадий:
1. **build**: Собирает бинарный файл `freq` в окружении `debian:11` с использованием CMake и сохраняет его как артефакт.
2. **test**: Запускает Python-скрипт `test_freq.py`, который проверяет корректность работы собранного бинарника на тестовых данных.
3. **docker**: Собирает Docker-образ приложения. При необходимости (раскомментировав строки) образ может быть автоматически загружен в Docker Registry компании.
